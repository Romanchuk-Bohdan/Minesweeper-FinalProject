# Гра "Сапер" (Minesweeper)

Це професійна реалізація класичної гри "Сапер", написана на мові C# (Windows Forms). Проєкт розроблено з акцентом на чисту архітектуру, застосування патернів проєктування GoF, принципів SOLID та сучасних технік рефакторингу для забезпечення високої якості коду та стабільної роботи.

---

## 1. Інтерфейс користувача (UI)

**Головне меню:**

- **Персоналізація:** Введення імені гравця для ведення індивідуальної статистики.
- **Налаштування:** Вибір рівня складності (Легкий, Середній, Складний) з анімованою панеллю.
- **Розумний старт:** Кнопка початку нової гри або продовження раніше збереженої сесії.

**Ігровий екран:**

- **Графіка:** Динамічне ігрове поле з авторськими текстурами клітинок, мін та прапорців.
- **Інформаційна панель:** Таймер реального часу, лічильник мін, відсоток відкриття поля та найкращий результат профілю.
- **Кнопка "Безпечне відкриття":** Інтелектуальна підказка, що підсвічує гарантовано безпечну клітинку на полі.
- **Меню паузи (Esc):** Можливість призупинити процес, зберегти стан гри та вийти.

**Екран завершення гри:**

- **Повідомлення:** Діалогові вікна з результатом (перемога чи поразка).
- **Відеоповтор:** Кнопка для запуску системи покрокового відтворення останньої гри.
- **Навігація:** Можливість почати реванш або перейти до наступного рівня складності.

---

## 2. Ігрова логіка

**Генерація поля:**

- Динамічне створення розміру сітки та кількості мін відповідно до обраної складності.
- **Алгоритм First Click:** Розстановка мін відбувається лише після першого кліку гравця, що виключає можливість програшу на першому ж ходу.

**Механіка взаємодії:**

- Відкриття клітинок (лівий клік) та встановлення прапорців (правий клік).
- Алгоритм автоматичного рекурсивного відкриття порожніх областей (Flood Fill).
- Логіка підказок, що використовує стан ігрової дошки для пошуку безпечних координат.

**Завершення гри:**

- Логіка моментальної детонації при потраплянні на міну та розкриття всіх небезпечних об'єктів.
- Перевірка умови перемоги при відкритті всіх безпечних клітинок поля.
- Автоматичне порівняння результату з рекордом профілю та оновлення статистики.

---

## 3. Збереження даних

**Локальні файли (JSON):**

- **Профілі:** Збереження статистики ігор, перемог та персональних рекордів часу у файлі `profiles.json`.
- **Сесії:** Збереження повного стану ігрової дошки, таймера та розстановки мін (`GameMemento`) для відновлення гри.
- **Автоматизація:** Дані завантажуються через `ProfileManager` при старті та автоматично оновлюються або видаляються після фіналу партії.

---

## 4. Додаткові можливості

- **Система відеоповторів (Replay System):** Повноцінна реалізація на базі патерну Command, що дозволяє покроково відтворити ігрову сесію для аналізу стратегії.
- **Інтелектуальний помічник (Safe Cell):** Алгоритмічний пошук вільної клітинки, що дозволяє уникнути ситуацій "вгадування" на складних рівнях.
- **Анімований інтерфейс:** Використання таймерів для плавної анімації розгортання панелі налаштувань.
- **Авторський графічний дизайн:** Унікальні візуальні ресурси, створені спеціально для цього проєкту.

## 6. Design Patterns (Патерни проєктування)

### 1. **Singleton** — `Core/ProfileManager.cs`

Використовується для глобального доступу до системи профілів та цілісності файлових операцій.

```csharp
public static ProfileManager Instance => _instance ??= new ProfileManager();
```

### 2. **Memento (Зберігач)** — `Core/GameMemento.cs`

Дозволяє створювати "знімки" стану поля без порушення інкапсуляції об'єкта `Game`.

### 3. **Command (Команда)** — `Core/Commands.cs`

Інкапсулює дії гравця (Reveal/Flag) в об'єкти, що є основою роботи системи повторів (Replay).

```csharp
public class RevealCommand : ICommand
{
    public List<Cell> Execute() => _game.RevealCell(Row, Col);
}
```

## 7. Refactoring Techniques (Техніки рефакторингу)

### 1. **Extract Class**

Логіку роботи з часом винесено в окремий клас `TimeManager.cs`.

### 2. **Replace Magic Numbers with Constants**

Заміна числових значень розмірів та анімації на іменовані константи (`GamePanelDefaultWidth`, `MenuPanelMaxHeight`).

### 3. **Extract Method**

Розбиття методу `StartGame()` на логічні частини для кращої читабельності:

```csharp
private void StartGame()
{
    InitializeGameData();
    ConfigureUIForGame();
    UpdateGameStatsUI();
}
```

### 4. **Decompose Conditional**

Логіка обробки фіналу гри винесена в асинхронні методи `HandleGameWinAsync()` та `HandleGameLossAsync()`.

### 5. **Encapsulate Collection**

Захист колекції профілів від несанкціонованих змін через `IReadOnlyList`.

```csharp
public IReadOnlyList<Profile> GetAllProfiles() => _profiles.AsReadOnly();
```

## 8. Programming Principles (Принципи програмування)

1. **SRP (Single Responsibility):** Кожен клас відповідає за свою область (час, збереження, малювання).

2. **OCP (Open/Closed):** Система команд дозволяє додавати нові типи ходів без зміни менеджера повторів.

3. **DRY (Don't Repeat Yourself):** Спільна логіка отримання сусідів клітинки реалізована в методі `Board.GetNeighbors`.

4. **KISS (Keep It Simple):** Використання зрозумілих циклів `foreach` та структур замість переускладненого коду.

5. **Fail Fast:** Валідація імені профілю та стану файлів одразу при запуску відповідних методів.
